# Configuración de Parámetros MySQL/MariaDB

Esta tabla muestra los parámetros de configuración, los valores asignados y una breve explicación del porqué se eligieron dichos valores.

<table>
  <thead>
    <tr>
      <th>Parámetro</th>
      <th>Valor</th>
      <th>Explicación</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>innodb_buffer_pool_size</td>
      <td>4G</td>
      <td>Tamaño de la pool de buffer para manejar grandes cantidades de datos en memoria.</td>
    </tr>
    <tr>
      <td>innodb_log_file_size</td>
      <td>512M</td>
      <td>Ajustado para reducir la frecuencia de escrituras y mejorar el rendimiento.</td>
    </tr>
    <tr>
      <td>max_connections</td>
      <td>200</td>
      <td>Se espera una alta concurrencia de conexiones.</td>
    </tr>
    <tr>
      <td>query_cache_size</td>
      <td>64M</td>
      <td>Tamaño adecuado para mejorar el rendimiento de consultas repetidas.</td>
    </tr>
    <tr>
      <td>table_open_cache</td>
      <td>1024</td>
      <td>Aumentado para mejorar la eficiencia en servidores con muchas tablas.</td>
    </tr>
    <tr>
      <td>tmp_table_size</td>
      <td>128M</td>
      <td>Para evitar la creación de tablas temporales en disco.</td>
    </tr>
    <tr>
      <td>max_heap_table_size</td>
      <td>128M</td>
      <td>Similar al valor de `tmp_table_size` para manejar mejor las tablas en memoria.</td>
    </tr>
    <tr>
      <td>innodb_flush_log_at_trx_commit</td>
      <td>1</td>
      <td>Garantiza integridad en cada transacción (ideal para ambientes críticos).</td>
    </tr>
    <tr>
      <td>log_bin</td>
      <td>ON</td>
      <td>Activado para habilitar la replicación y el punto de recuperación de datos.</td>
    </tr>
    <tr>
      <td>slow_query_log</td>
      <td>ON</td>
      <td>Activado para identificar consultas lentas y optimizarlas.</td>
    </tr>
    <tr>
      <td>slow_query_log_file</td>
      <td>/var/log/mysql/slow.log</td>
      <td>Archivo donde se almacenan las consultas lentas.</td>
    </tr>
    <tr>
      <td>long_query_time</td>
      <td>2</td>
      <td>Tiempo en segundos para definir consultas lentas.</td>
    </tr>
    <tr>
      <td>bind-address</td>
      <td>0.0.0.0</td>
      <td>Escucha en todas las interfaces de red.</td>
    </tr>
    <tr>
      <td>innodb_file_per_table</td>
      <td>ON</td>
      <td>Cada tabla tendrá su propio archivo de datos, mejorando la gestión.</td>
    </tr>
    <tr>
      <td>performance_schema</td>
      <td>OFF</td>
      <td>Desactivado para evitar sobrecarga en el rendimiento del servidor.</td>
    </tr>
  </tbody>
</table>
